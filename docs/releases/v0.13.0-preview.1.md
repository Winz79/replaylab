# ReplayLab v0.13.0-preview.1 — SDK Packaging, Hostable Surfaces, and Release Automation

> **Note:** This release has not been published yet. The tag must be created and pushed manually after this PR is merged.

## Highlights

- **Packageable ReplayLab SDK** — Core, parsers, adapters, and hosting libraries are now packageable and consumable via `PackageReference`.
- **CLI/Web/Desktop hostable surfaces** — Private projects can own composition roots and host ReplayLab surfaces without modifying the public repository.
- **External-style CustomReplayTool sample** — Demonstrates consuming ReplayLab through local NuGet packages with custom parser and sender composition.
- **Parser/sender composition hardening** — `TryAdd*` conventions documented and tested so consumer registrations take precedence.
- **GitHub Packages release automation** — Pushing a version tag triggers restore, build, test, pack, and publish to GitHub Packages with an attached GitHub Release.

## How to consume packages

Add the GitHub Packages NuGet source:

```powershell
dotnet nuget add source "https://nuget.pkg.github.com/sebastienwitz/index.json" `
  --name github-replaylab `
  --username <github-username> `
  --password <github-token> `
  --store-password-in-clear-text
```

Then reference packages in your project (after the release exists):

```xml
<PackageReference Include="ReplayLab.Core" Version="0.13.0-preview.1" />
```

> **Note:** GitHub Packages may require authentication. Use a personal access token with `read:packages` scope for consuming packages.

## Verification before tagging

Run these checks before creating the release tag:

```powershell
git fetch --tags
git tag --sort=version:refname
gh release list

dotnet restore ReplayLab.sln
dotnet build ReplayLab.sln --configuration Release --no-restore
dotnet test ReplayLab.sln --configuration Release --no-build
node --check src/ReplayLab.Web.Hosting/wwwroot/js/replay-grid.js

pwsh ./eng/pack-local.ps1
pwsh ./eng/verify-local-packages.ps1
pwsh ./eng/verify-custom-replay-tool-sample.ps1
```
