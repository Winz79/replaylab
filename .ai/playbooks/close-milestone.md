# Close Milestone Playbook

Use this when closing a milestone after implementation work is complete.

## Confirm Scope

1. Review milestone goals, issues, PRs, and acceptance criteria.
2. Confirm completed work against the milestone plan.
3. List deferred, dropped, or changed scope explicitly.
4. Check that docs and ADRs reflect final decisions.

## Release Steps

- Create or confirm the release tag when requested by the repository owner.
- Draft release notes from merged PRs and milestone outcomes.
- Include verification evidence, known limitations, and upgrade notes when relevant.
- Do not tag or publish a release unless explicitly asked.

## Retrospective

Capture:

- what shipped
- what changed from the plan
- key decisions made
- risks that remain
- follow-up issues or issue drafts
- process improvements for the next milestone

## Roadmap Update

- Update the roadmap or next milestone plan when requested or when closure changes sequencing.
- Keep updates factual and brief.
- Do not modify PRDs or ADRs unless closure reveals a decision or outcome they must reflect.
