# Create Issue From Plan Playbook

Use this when creating GitHub issues from reviewed planning artifacts or issue drafts.

## Inputs

- milestone plan
- PRD-light
- ADRs or ADR candidates
- reviewed issue drafts
- target GitHub milestone

## Rules

- Do not create issues until explicitly asked.
- Create issues only from reviewed drafts.
- Keep issues small and vertical-slice oriented.
- Do not create issues for future milestones unless explicitly requested.
- Do not duplicate existing open issues.
- Assign the correct milestone when known.
- Apply relevant labels when available.

## Issue Body

Each issue should include:

- goal
- source docs
- scope
- out of scope
- acceptance criteria
- verification
- risks / assumptions

## After Creation

Report:

- created issue numbers
- target milestone
- skipped drafts and why
- assumptions or missing metadata
